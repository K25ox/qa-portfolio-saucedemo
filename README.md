# QA Portfolio - SauceDemo

This project showcases my manual QA testing skills using the SauceDemo web application.

## 📌 Project Overview
The objective of this project is to simulate real-world QA activities, including test case design, test execution, and bug reporting.

## 🧪 Scope
The following functionalities were tested:
- Login
- Product selection
- Cart functionality
- Checkout process

## 🛠️ Testing Activities
- Test case design and documentation
- Manual test execution
- Bug identification and reporting

## 📂 Repository Structure
- `/test-cases` → Test cases for login and related scenarios
- `/bug-reports` → Documented defects found during testing

## 🔍 Key Findings

During the testing process, several issues were identified across different areas of the application:

- **Checkout validation issue:** The system allows users to proceed with invalid or incomplete purchase data (e.g., invalid postal code).
- **Cart validation issue:** Users are able to continue the checkout process with an empty cart.
- **Error message clarity issue:** Some validation messages are unclear or contain non-professional wording (e.g., "Epic sadface").

## 🧠 Testing Approach

The testing approach focused on:

- Functional testing of critical user flows (login, cart, checkout)
- Validation of input fields and business rules
- Negative testing (invalid data, empty fields)
- Basic UX validation (error messages and user feedback)

## 📌 Conclusion

This project demonstrates the ability to design test cases, identify defects, and document findings in a clear and structured way, simulating a real QA workflow.

# QA Portfolio - SauceDemo

Este proyecto muestra mis habilidades en testing manual QA utilizando la aplicación web SauceDemo.

## 📌 Descripción del Proyecto
El objetivo de este proyecto es simular actividades reales de QA, incluyendo el diseño de casos de prueba, ejecución de pruebas y reporte de errores.

## 🧪 Alcance
Las siguientes funcionalidades fueron testeadas:
- Login
- Selección de productos
- Funcionalidad del carrito
- Proceso de checkout

## 🛠️ Actividades de Testing
- Diseño y documentación de casos de prueba
- Ejecución manual de pruebas
- Identificación y reporte de bugs

## 📂 Estructura del Repositorio
- `/test-cases` → Casos de prueba para login y escenarios relacionados
- `/bug-reports` → Defectos documentados encontrados durante el testing

## 🔍 Hallazgos Clave

Durante el proceso de testing, se identificaron varios problemas en diferentes áreas de la aplicación:

- **Problema de validación en checkout:** El sistema permite avanzar con datos inválidos o incompletos (por ejemplo, código postal inválido).
- **Problema en el carrito:** El usuario puede continuar el proceso de compra con el carrito vacío.
- **Problema de claridad en mensajes de error:** Algunos mensajes no son claros o contienen texto poco profesional (por ejemplo, "Epic sadface").

## 🧠 Enfoque de Testing

El enfoque de testing se centró en:

- Pruebas funcionales de flujos críticos (login, carrito, checkout)
- Validación de campos y reglas de negocio
- Pruebas negativas (datos inválidos, campos vacíos)
- Validación básica de experiencia de usuario (mensajes de error y feedback)

## 📌 Conclusión

Este proyecto demuestra la capacidad de diseñar casos de prueba, identificar defectos y documentar hallazgos de manera clara y estructurada, simulando un flujo de trabajo real de QA.
