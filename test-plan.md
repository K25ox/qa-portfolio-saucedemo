# Test Plan - SauceDemo

## 1. Objective

The objective of this test plan is to validate the main functionalities of the SauceDemo application, ensuring that critical user flows such as login, cart, and checkout work correctly and meet expected behavior.

## 2. Scope

### In Scope
The following functionalities will be tested:
- Login functionality
- Product listing
- Add to cart
- Cart functionality
- Checkout process

### Out of Scope
The following areas are not covered in this testing:
- Performance testing
- Security testing
- Mobile testing
- Cross-browser testing

## 3. Testing Approach

The testing approach for this project includes:

- Manual functional testing of core user flows (login, cart, checkout)
- Positive testing (valid inputs and expected behavior)
- Negative testing (invalid data, empty fields, incorrect inputs)
- Validation of business rules (e.g., cart cannot be empty during checkout)
- Basic usability testing focused on error messages and user feedback

## 4. Test Scenarios

The following test scenarios were covered:

- Login with valid credentials
- Login with invalid password
- Login with empty username field
- Add products to cart
- Access cart with and without products
- Proceed to checkout with valid data
- Attempt checkout with invalid or missing data
- Attempt checkout with empty cart

## 5. Entry and Exit Criteria

### Entry Criteria
- The application is accessible
- The test environment is available
- Test data (valid user credentials) is available

### Exit Criteria
- All planned test scenarios have been executed
- All identified bugs have been documented
- Critical defects have been reported

# Plan de Pruebas - SauceDemo

## 1. Objetivo

El objetivo de este plan de pruebas es validar las funcionalidades principales de la aplicación SauceDemo, asegurando que los flujos críticos de usuario como login, carrito y checkout funcionen correctamente y cumplan con el comportamiento esperado.

## 2. Alcance

### Dentro del Alcance (In Scope)
Las siguientes funcionalidades serán testeadas:
- Funcionalidad de login
- Listado de productos
- Agregado de productos al carrito
- Funcionalidad del carrito
- Proceso de checkout

### Fuera del Alcance (Out of Scope)
Las siguientes áreas no están contempladas en este testing:
- Pruebas de rendimiento
- Pruebas de seguridad
- Pruebas en dispositivos móviles
- Pruebas cross-browser

## 3. Enfoque de Testing

El enfoque de testing para este proyecto incluye:

- Pruebas funcionales manuales de los flujos principales (login, carrito, checkout)
- Pruebas positivas (datos válidos y comportamiento esperado)
- Pruebas negativas (datos inválidos, campos vacíos, entradas incorrectas)
- Validación de reglas de negocio (por ejemplo, el carrito no debe estar vacío durante el checkout)
- Pruebas básicas de usabilidad enfocadas en mensajes de error y feedback al usuario

## 4. Escenarios de Prueba

Se cubrieron los siguientes escenarios de prueba:

- Login con credenciales válidas
- Login con contraseña inválida
- Login con campo de usuario vacío
- Agregar productos al carrito
- Acceder al carrito con y sin productos
- Avanzar al checkout con datos válidos
- Intentar checkout con datos inválidos o incompletos
- Intentar checkout con el carrito vacío

## 5. Criterios de Entrada y Salida

### Criterios de Entrada
- La aplicación se encuentra accesible
- El entorno de pruebas está disponible
- Se cuenta con datos de prueba (credenciales válidas)

### Criterios de Salida
- Todos los escenarios de prueba planificados han sido ejecutados
- Todos los bugs identificados han sido documentados
- Los defectos críticos han sido reportados
