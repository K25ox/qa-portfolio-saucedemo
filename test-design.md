# Test Design Techniques

## Equivalence Partitioning
Test cases were designed by grouping input data into valid and invalid partitions, reducing the number of test cases while maintaining coverage.

## Boundary Value Analysis
Boundary values were tested to verify system behavior at the limits of input ranges.

## Edge Cases
Uncommon and unexpected scenarios were tested to identify potential system failures and improve robustness.

# Técnicas de Diseño de Pruebas

## Partición de Equivalencia
Los casos de prueba fueron diseñados agrupando los datos de entrada en particiones válidas e inválidas, reduciendo la cantidad de pruebas necesarias sin perder cobertura.

## Análisis de Valores Límite
Se probaron los valores en los límites para verificar el comportamiento del sistema en los extremos de los rangos de entrada.

## Casos Extremos (Edge Cases)
Se probaron escenarios poco comunes o inesperados para identificar posibles fallas del sistema y mejorar su robustez.
